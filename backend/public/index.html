02 / About

ABOUT

I'm an engineer who treats AI systems like products — not demos. My work sits at the intersection of agentic AI, automation and backend engineering, turning language models into things that actually finish a job.

04 Core disciplines
Agents · GenAI · Backend · Cloud

"Most AI demos stop at the chat window. I build the layer underneath — the orchestration, memory, and tooling that make an agent reliable enough to ship."

The journey

I started as a software developer solving conventional backend problems, then moved toward generative AI as the tooling matured — LangChain, LangGraph, vector retrieval, and multi-agent orchestration became my daily stack. Today I design systems where multiple agents reason, call tools, and recover from their own mistakes.

Engineering mindset

I think in systems, not features. Every agent I build is designed around failure modes first: what happens when the model hallucinates, when an API times out, when the user gives an ambiguous instruction. Resilience is the unglamorous 80% of agentic engineering.

04 / Selected Work

PROJECTS

Case Study

MULTI-AGENT DATABASE ASSISTANT

Problem

Non-technical teams couldn't query production databases without writing SQL, creating a bottleneck on engineering for every ad-hoc data request.

Solution

A multi-agent system where a planner agent decomposes natural-language questions, a SQL agent writes and validates queries, and a critic agent checks results before they reach the user.

Architecture

LangGraph orchestration with isolated tool-calling agents, a schema-aware retrieval layer, and a sandboxed query executor with automatic rollback on unsafe statements.

Tech Stack
PythonLangGraphFastAPIPostgreSQLDocker
Impact

85% reduction in ad-hoc SQL requests routed to engineering; non-technical teams now self-serve data answers in seconds.
